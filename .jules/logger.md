## 2024-05-24 - Wrapped unconditional logs and warned silent exceptions
**Learning:** Found several unconditional `Log.Message` calls and silent `catch { }` and `catch (Exception)` blocks in `Source/AsyncAIClient.cs`, `Source/UI_AIPortraitCard.cs`, `Source/ModSettings.cs`, `Source/CacheManager.cs`, `Source/PawnStateExtractor.cs`, and `Source/ModController.cs`. Spammed logs hide real errors and silent exceptions can mask problems like missing features or cache read/write issues.
**Action:** Always wrap standard output logs like `Log.Message` in `if (Prefs.DevMode)` so they are hidden from normal users but visible to developers. Replace empty/silent exception catches with a logged warning, also wrapped in `if (Prefs.DevMode)`, to keep production logs clean but retain debug context. Note that some silent catch blocks designed to safely handle backward compatibility (like missing psychic amplifiers) can be left alone, but should at least have a comment explaining why.
