## 2024-05-19 - Removed GC allocations in UI hot paths
**Learning:** Found massive allocation vectors hiding in `OnGUI` UI loops. `UI_AIPortraitCard.GetPortraitTexture` concatenated string keys (like `pawn.ThingID + "_" + framing`) multiple times per frame, boxing `int` to `string` repeatedly for multiple UI queries. It also allocated an array every frame (`new[] { "portrait", "bodyshot", "special" }`) and used a `foreach` loop over it. Additionally, `UI_StoryBook.DoWindowContents` contained multiple `foreach` loops which allocated enumerators per frame.
**Action:** Replaced string concatenation keys with a struct-based key (`PawnFramingKey`) across all tracking dictionaries in `AIPortraitsManager` to avoid per-frame allocations. Replaced the `new[]` array allocation with a `static readonly string[]` cache, and converted all `foreach` loops in `UI_StoryBook.DoWindowContents` and `GetPortraitTexture` to `for` loops.
