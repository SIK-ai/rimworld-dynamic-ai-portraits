## 2024-05-24 - Inline Validation and Async Feedback
**Learning:** For inputs like API keys, adding inline validation with specific styling (e.g., red text warning) below the input field makes the UI more robust and prevents errors later on. For disabled states (like a disabled generate button), showing an hourglass state or adding tooltips significantly improves clarity for users.
**Action:** Always include inline validation messages for required form fields. Additionally, provide clear context for disabled buttons using tooltips or text change.
